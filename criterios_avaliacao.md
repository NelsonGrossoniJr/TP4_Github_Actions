# 4. Refatorar código e implementar esteiras de CI/CD eficientes, utilizando GitHub Actions

## Critérios de Avaliação

1. O aluno identificou e eliminou problemas no código, como redundâncias, métodos longos e classes sobrecarregadas, promovendo clareza e eficiência?

2. O aluno refatorou o código guiado por testes, simplificando funções, eliminando redundâncias e mantendo o comportamento consistente?

3. O aluno criou e configurou workflows no GitHub Actions, organizando jobs e steps para automação de build, teste e validação?

4. O aluno integrou actions do GitHub Marketplace, utilizando-as para otimizar processos de automação e facilitar a execução de workflows?

5. O aluno gerenciou runners no GitHub Actions, configurando ambientes personalizados para execução eficiente e segura de jobs?

6. O aluno entregou o artigo com Resultados e Discussão e Conclusão?

7. O aluno implementou esteiras de CI/CD eficientes, com gatilhos automatizados e pipelines claros para build, testes e integração contínua?
